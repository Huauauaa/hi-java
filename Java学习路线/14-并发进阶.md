# 第 14 章：并发进阶

## 学习目标

- 掌握 Lock、线程池与并发集合
- 了解 JUC 包核心工具

## 核心知识点

### Lock 接口

```java
ReentrantLock lock = new ReentrantLock();
lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();
}
```

- vs `synchronized`：可中断、可超时、公平锁、多 Condition
- `ReadWriteLock`：读写分离，`ReentrantReadWriteLock`

### 线程池

```java
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> doWork());
pool.shutdown();
```

| 类型 | 特点 |
|------|------|
| FixedThreadPool | 固定线程数 |
| CachedThreadPool | 可缓存，适合短任务 |
| SingleThreadExecutor | 单线程顺序 |
| ScheduledThreadPool | 定时/周期 |

- 生产环境推荐 `ThreadPoolExecutor` 手动配置核心参数
- 参数：corePoolSize、maximumPoolSize、queue、RejectedExecutionHandler

### JUC 并发工具

| 类 | 用途 |
|----|------|
| `CountDownLatch` | 等待 N 个任务完成 |
| `CyclicBarrier` | N 线程互相等待 |
| `Semaphore` | 限流许可 |
| `ConcurrentHashMap` | 并发 Map |
| `CopyOnWriteArrayList` | 读多写少 |

### 原子类

- `AtomicInteger`、`AtomicReference`、`LongAdder`
- CAS 无锁更新

### volatile

- 保证可见性与禁止指令重排（不保证原子性）
- 适用：状态标志、双重检查锁的单例

### 并发设计原则

- 缩小锁粒度、减少锁持有时间
- 优先不可变与线程安全容器
- 避免死锁：固定加锁顺序

## 重点难点

- 线程池队列满时的拒绝策略
- `ConcurrentHashMap` JDK 7 vs 8 实现差异
