# 第 08 章：抽象类与接口

## 学习目标

- 区分抽象类与接口的使用场景
- 掌握 Java 8+ 接口默认方法与静态方法

## 核心知识点

### 抽象类

```java
public abstract class Shape {
    abstract double area();

    void describe() {  // 可有具体方法
        System.out.println("This is a shape.");
    }
}
```

- 不能实例化，可含抽象方法与具体方法
- 可有构造方法、成员变量
- 适合「is-a」且共享实现的层次

### 接口

```java
public interface Flyable {
    void fly();                          // 默认 public abstract

    default void land() {                // Java 8+
        System.out.println("Landing...");
    }

    static void info() {                 // Java 8+
        System.out.println("Flyable interface");
    }
}
```

- 多实现：`class Bird implements Flyable, Runnable`
- 成员默认 `public static final`（常量）
- Java 8+：`default`、`static` 方法
- Java 9+：接口可有 `private` 方法

### 抽象类 vs 接口

| | 抽象类 | 接口 |
|---|--------|------|
| 继承/实现 | 单继承 | 多实现 |
| 构造方法 | 有 | 无 |
| 成员变量 | 任意 | 常量 |
| 设计意图 | 模板 + 部分实现 | 能力契约 |

### 函数式接口

- 仅一个抽象方法的接口
- `@FunctionalInterface` 注解
- 为 Lambda 表达式铺垫（见第 17 章）

### 内部类（延伸）

- 成员内部类、静态内部类、局部内部类、匿名内部类
