# 第 16 章：反射与注解

## 学习目标

- 使用反射在运行时 inspect 与操作类
- 理解注解的定义与处理

## 核心知识点

### 反射（Reflection）

```java
Class<?> clazz = Class.forName("com.example.User");
// 或 User.class、obj.getClass()

Constructor<?> ctor = clazz.getDeclaredConstructor(String.class);
Object obj = ctor.newInstance("Alice");

Method method = clazz.getMethod("getName");
String name = (String) method.invoke(obj);

Field field = clazz.getDeclaredField("age");
field.setAccessible(true);  // 突破 private
field.setInt(obj, 20);
```

### Class 对象

- 每个类加载后对应一个 `Class` 实例
- 获取：`.class`、`getClass()`、`Class.forName()`

### 反射应用场景

- 框架：Spring 依赖注入、MyBatis 映射
- 动态代理、序列化、测试工具

### 注解（Annotation）

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyAnnotation {
    String value() default "";
    int priority() default 0;
}
```

### 元注解

| 注解          | 作用                           |
| ------------- | ------------------------------ |
| `@Target`     | 可用于哪些元素                 |
| `@Retention`  | 保留策略：SOURCE/CLASS/RUNTIME |
| `@Documented` | 出现在 Javadoc                 |
| `@Inherited`  | 子类继承                       |

### 常见内置注解

- `@Override`、`@Deprecated`、`@SuppressWarnings`
- `@FunctionalInterface`

### 注解处理

- 编译期：注解处理器（APT）
- 运行期：反射读取 `@Retention(RUNTIME)`

```java
if (method.isAnnotationPresent(MyAnnotation.class)) {
    MyAnnotation ann = method.getAnnotation(MyAnnotation.class);
}
```

## 重点难点

- 反射破坏封装，性能低于直接调用
- 框架大量使用反射 + 注解，理解原理有助于调试
