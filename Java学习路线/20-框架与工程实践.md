# 第 20 章：框架与工程实践

## 学习目标

- 了解 Java 企业级生态与主流框架
- 掌握工程化开发的基本流程与工具

## 核心知识点

### 构建工具

| 工具   | 特点                    |
| ------ | ----------------------- |
| Maven  | XML 配置，生态成熟      |
| Gradle | Groovy/Kotlin DSL，灵活 |

- 依赖管理、生命周期、多模块项目
- 坐标：`groupId:artifactId:version`

### 单元测试

```java
@Test
void shouldReturnSum() {
    assertEquals(3, calculator.add(1, 2));
}
```

- JUnit 5、Mockito、AssertJ
- 测试金字塔：单元 > 集成 > E2E

### Spring 框架

| 模块        | 作用               |
| ----------- | ------------------ |
| Spring Core | IoC 容器、依赖注入 |
| Spring MVC  | Web 层 REST API    |
| Spring Boot | 自动配置、快速启动 |
| Spring Data | 数据访问抽象       |

```java
@RestController
@RequestMapping("/users")
public class UserController {
    @Autowired
    private UserService userService;

    @GetMapping("/{id}")
    public User get(@PathVariable Long id) {
        return userService.findById(id);
    }
}
```

### 持久层

- JDBC → MyBatis（SQL 映射）→ JPA/Hibernate（ORM）
- 连接池：HikariCP
- 事务：`@Transactional`

### Web 与微服务

- Servlet / Jakarta EE 基础
- RESTful API 设计
- Spring Cloud：注册发现、配置中心、网关、熔断（Nacos、Gateway、Sentinel）

### 日志

- SLF4J 门面 + Logback / Log4j2
- 级别：TRACE < DEBUG < INFO < WARN < ERROR

### 其他工程实践

| 主题     | 要点                                     |
| -------- | ---------------------------------------- |
| Git      | 分支策略、PR、代码审查                   |
| CI/CD    | GitHub Actions、Jenkins                  |
| 代码规范 | Checkstyle、SpotBugs、阿里巴巴 Java 规范 |
| 文档     | Javadoc、OpenAPI/Swagger                 |
| 容器化   | Docker 打包 JAR/WAR                      |

### 学习路径建议

```
Java 基础（01–12）
    → 并发与高级特性（13–17）
    → JVM + 设计模式（18–19）
    → Spring Boot 小项目
    → 数据库 + MyBatis/JPA
    → 分布式与中间件（Redis、MQ、Kafka）
```

### 推荐练手项目

- 待办事项 REST API（Spring Boot + JPA）
- 博客系统（用户、文章、评论）
- 简易电商（商品、订单、支付 mock）

## 结语

前 19 章打好语言与计算机基础，本章衔接工业界真实开发。持续写代码、读源码、做小项目，比单纯看知识点更重要。
