# 第 10 章：异常处理

## 学习目标

- 理解异常体系与分类
- 正确使用 try-catch-finally 与 throws

## 核心知识点

### 异常体系

```
Throwable
├── Error（JVM 级，一般不捕获）
│   └── OutOfMemoryError, StackOverflowError
└── Exception
    ├── RuntimeException（非受检）
    │   └── NPE, IllegalArgumentException, IndexOutOfBoundsException
    └── 其他 Exception（受检）
        └── IOException, SQLException
```

### 受检 vs 非受检

|        | 受检异常                  | 非受检异常         |
| ------ | ------------------------- | ------------------ |
| 继承   | `Exception`（非 Runtime） | `RuntimeException` |
| 编译器 | 必须处理或声明            | 不强制             |
| 典型   | IO、SQL                   | 编程错误           |

### 处理语法

```java
try {
    // 可能抛异常的代码
} catch (IOException e) {
    e.printStackTrace();
} catch (Exception e) {
    // 子类 catch 在前
} finally {
    // 几乎总是执行，用于释放资源
}
```

### try-with-resources（Java 7+）

```java
try (InputStream in = new FileInputStream("a.txt")) {
    // 使用 in
}  // 自动 close，实现 AutoCloseable
```

### throws 与 throw

- `throws`：方法签名声明可能抛出的异常
- `throw`：主动抛出异常对象

### 自定义异常

```java
public class BusinessException extends RuntimeException {
    public BusinessException(String message) {
        super(message);
    }
}
```

### 最佳实践

- 不要吞掉异常（空 catch）
- 捕获具体类型，避免裸 `Exception`
- 业务异常用非受检，可恢复错误用受检
- 日志记录异常栈，不要只 `printStackTrace`

- 设计业务异常类，统一错误码与消息
