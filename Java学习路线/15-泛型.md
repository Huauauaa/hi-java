# 第 15 章：泛型

## 学习目标

- 理解泛型的作用与类型擦除
- 掌握通配符与边界

## 核心知识点

### 为什么需要泛型

- 编译期类型检查，减少强转
- 代码复用，类型安全

```java
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(123);  // 编译错误
```

### 泛型类与方法

```java
public class Box<T> {
    private T value;
    public T get() { return value; }
    public void set(T value) { this.value = value; }
}

public static <T> T first(List<T> list) {
    return list.isEmpty() ? null : list.get(0);
}
```

### 通配符

| 写法            | 含义                           |
| --------------- | ------------------------------ |
| `<?>`           | 未知类型                       |
| `<? extends T>` | 上界，只读（Producer extends） |
| `<? super T>`   | 下界，可写（Consumer super）   |

```java
void copy(List<? extends Number> src, List<? super Number> dest) { /* ... */ }
```

### 类型擦除

- 编译后泛型信息被擦除为边界或 Object
- `List<String>` 与 `List<Integer>` 运行时都是 `List`
- 不能 `new T()`、不能 `instanceof T`（直接）

### 泛型限制

- 不能创建泛型数组 `new T[10]`
- 静态字段类型不能是类型参数
- 不能 catch 泛型异常类型

### PECS 原则

- **P**roducer **E**xtends：生产者用 extends
- **C**onsumer **S**uper：消费者用 super

## 实践建议

- 阅读 `Collections.sort`、`Comparator.comparing` 的泛型签名
- 实现简单的泛型栈 `Stack<T>`
