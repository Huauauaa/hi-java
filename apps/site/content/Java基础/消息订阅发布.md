# 消息订阅发布

> **发布-订阅（Pub/Sub）**：发布者只负责发消息，不关心谁收；订阅者只关心自己感兴趣的主题，在消息到达时被通知。两者通过**主题（Topic）/ 事件（Event）**解耦。

## 与点对点对比

| 模式       | 关系           | 典型场景                     |
| ---------- | -------------- | ---------------------------- |
| 点对点     | 1 发 → 1 收    | 任务队列、RPC 调用           |
| 发布-订阅  | 1 发 → N 收    | 配置变更广播、领域事件、日志 |

## 应用内：观察者模式

Java 最直观的 Pub/Sub 实现：定义事件类型，维护订阅者列表，发布时逐个回调。

```java
// 事件
record OrderPaidEvent(String orderId, int amount) {}

// 订阅者
interface EventListener<T> {
    void onEvent(T event);
}

// 发布者（简易 EventBus）
class EventBus {
    private final Map<Class<?>, List<EventListener<?>>> listeners = new HashMap<>();

    public <T> void subscribe(Class<T> type, EventListener<T> listener) {
        listeners.computeIfAbsent(type, k -> new ArrayList<>()).add(listener);
    }

    @SuppressWarnings("unchecked")
    public <T> void publish(T event) {
        List<EventListener<?>> ls = listeners.getOrDefault(event.getClass(), List.of());
        for (EventListener<?> l : ls) {
            ((EventListener<T>) l).onEvent(event);
        }
    }
}
```

要点：

- 订阅按**事件类型**注册，发布时只通知匹配类型的监听器
- 监听器里避免耗时操作，否则会拖慢发布线程（异步见下文）
- `java.util.Observable` / `Observer` 已过时，新项目自己写或用框架

## Spring 事件

Spring 容器内自带 Pub/Sub：`ApplicationEventPublisher` 发布，`@EventListener` 订阅。

```java
// 事件
public class OrderPaidEvent extends ApplicationEvent {
    private final String orderId;

    public OrderPaidEvent(Object source, String orderId) {
        super(source);
        this.orderId = orderId;
    }

    public String getOrderId() {
        return orderId;
    }
}

// 发布
@Service
public class OrderService {
    private final ApplicationEventPublisher publisher;

    public OrderService(ApplicationEventPublisher publisher) {
        this.publisher = publisher;
    }

    public void pay(String orderId) {
        // ... 落库、改状态
        publisher.publishEvent(new OrderPaidEvent(this, orderId));
    }
}

// 订阅
@Component
public class InventoryListener {
    @EventListener
    public void onPaid(OrderPaidEvent event) {
        System.out.println("扣库存: " + event.getOrderId());
    }
}
```

- 默认在**发布线程**同步执行；加 `@Async` 可异步（需开启 `@EnableAsync`）
- 跨服务、跨进程不要用 Spring 事件，改用 MQ

## 分布式消息中间件

进程内 EventBus 无法跨 JVM。多服务之间用 **Broker** 做中转：

| 产品        | 模型简述                              |
| ----------- | ------------------------------------- |
| Redis       | `PUBLISH channel msg` / `SUBSCRIBE`   |
| RabbitMQ    | Exchange（fanout / topic）→ Queue     |
| Kafka       | Topic + Partition，高吞吐日志流       |
| RocketMQ    | Topic + Tag 过滤，阿里系常用          |

共性术语：

- **Producer / Publisher**：发消息
- **Consumer / Subscriber**：收消息
- **Topic / Channel**：消息分类
- **Consumer Group**（Kafka 等）：同组内一条消息只被一个消费者处理

```java
// RabbitMQ 概念示意（Spring AMQP）
// 发布：routingKey = "order.paid"
rabbitTemplate.convertAndSend("order.exchange", "order.paid", payload);

// 订阅：队列绑定到 exchange + routingKey
@RabbitListener(queues = "inventory.queue")
public void handle(OrderPaidMessage msg) { /* 扣库存 */ }
```

## 设计注意

1. **至少一次投递**：网络抖动可能导致重复，消费端要做**幂等**（唯一键、状态机）
2. **顺序**：同一 key 的消息如需有序，选分区 key 或单队列
3. **失败重试**：死信队列（DLQ）兜住反复失败的消息
4. **背压**：消费慢于生产时，靠队列堆积或限流，不能无限内存

PubSubTest.java

```java
package com.hua.demo;

import org.junit.Test;

import java.util.ArrayList;
import java.util.List;

public class PubSubTest {

    @Test
    public void test() {
        EventBus bus = new EventBus();
        List<String> logs = new ArrayList<>();

        bus.subscribe(OrderPaidEvent.class, e -> logs.add("inventory:" + e.orderId()));
        bus.subscribe(OrderPaidEvent.class, e -> logs.add("sms:" + e.amount()));

        bus.publish(new OrderPaidEvent("O-1001", 99));

        System.out.println(logs);
        // [inventory:O-1001, sms:99]
    }
}
```
