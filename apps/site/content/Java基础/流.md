# 流

> **流（Stream）**：按顺序读写数据的抽象；数据像水流一样从源头流向目的地，程序只关心「读/写」，不关心底层是文件、网络还是内存。

## 分类

| 维度 | 类型 | 说明 |
| ---- | ---- | ---- |
| 方向 | 输入流、输出流 | 读数据 / 写数据 |
| 数据 | 字节流、字符流 | 二进制字节 / Unicode 字符 |
| 角色 | 节点流、处理流 | 直接连数据源 / 包装其他流，增强功能 |

```
字节流：InputStream  ←读→  OutputStream  ←写→
字符流：Reader       ←读→  Writer         ←写→
```

字符流底层仍是字节，但多了编解码（如 UTF-8）；处理文本优先用字符流。

## 字节流

抽象基类：`java.io.InputStream` / `OutputStream`。

| 类 | 作用 |
| --- | --- |
| `FileInputStream` / `FileOutputStream` | 读写文件 |
| `ByteArrayInputStream` / `ByteArrayOutputStream` | 读写内存字节数组 |
| `BufferedInputStream` / `BufferedOutputStream` | 缓冲，减少磁盘/网络访问次数 |

```java
// 文件复制（字节流）
try (InputStream in = new FileInputStream("in.bin");
     OutputStream out = new FileOutputStream("out.bin")) {
    byte[] buf = new byte[8192];
    int n;
    while ((n = in.read(buf)) != -1) {
        out.write(buf, 0, n);
    }
}
```

`read()` 返回 `-1` 表示流已读完；`read(byte[])` 返回本次读到的字节数，同样 `-1` 表示结束。

## 字符流

抽象基类：`java.io.Reader` / `Writer`。适合文本。

| 类 | 作用 |
| --- | --- |
| `FileReader` / `FileWriter` | 读写文本文件（默认平台编码，不推荐） |
| `InputStreamReader` / `OutputStreamWriter` | 字节流 ↔ 字符流，可指定 `Charset` |
| `BufferedReader` / `BufferedWriter` | 缓冲 + `readLine()` / `newLine()` |

```java
// 按行读写文本（指定 UTF-8）
try (BufferedReader reader = new BufferedReader(
         new InputStreamReader(new FileInputStream("in.txt"), StandardCharsets.UTF_8));
     BufferedWriter writer = new BufferedWriter(
         new OutputStreamWriter(new FileOutputStream("out.txt"), StandardCharsets.UTF_8))) {
    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line);
        writer.newLine();
    }
}
```

`readLine()` 会去掉行尾 `\n` / `\r\n`，不会把换行符放进 `line`。

## 节点流与处理流

- **节点流**：直接对接数据源/目的地（文件、数组、管道等）
- **处理流**：包装已有流，叠加缓冲、压缩、加密、对象序列化等能力

典型组合：节点流 → 缓冲流 → 业务读写。

```java
// 处理流包装节点流
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("data.bin"));
```

`InputStream` 的装饰器链是 JDK 里装饰器模式的经典例子：`BufferedInputStream`、`DataInputStream` 等都可层层包装。

## 对象流（序列化）

`ObjectInputStream` / `ObjectOutputStream` 在字节流上读写对象。

```java
// 写
try (ObjectOutputStream oos = new ObjectOutputStream(
         new FileOutputStream("obj.bin"))) {
    oos.writeObject(user);
}

// 读
try (ObjectInputStream ois = new ObjectInputStream(
         new FileInputStream("obj.bin"))) {
    User user = (User) ois.readObject();
}
```

要求：

- 类实现 `Serializable`（标记接口）
- `transient` 字段不参与序列化
- 建议显式声明 `serialVersionUID`，避免类结构变更后反序列化失败

## 资源管理

流用完必须关闭，否则句柄泄漏。优先 **try-with-resources**（`AutoCloseable`）：

```java
try (InputStream in = new FileInputStream("a.txt")) {
  in.read();
} // 自动 close，即使抛异常也会先关流
```

旧写法需 `finally` 里手动 `close()`，且 `close` 本身也可能抛异常，容易写错。

## NIO.2 文件 API（补充）

日常读写小文件可直接用 `java.nio.file.Files`，内部已处理好流与编码：

```java
Path path = Paths.get("data.txt");
List<String> lines = Files.readAllLines(path, StandardCharsets.UTF_8);
Files.write(path, lines, StandardCharsets.UTF_8);
Files.copy(Paths.get("in.bin"), Paths.get("out.bin"));
```

大文件或需要精细控制缓冲时，仍用传统流或 `FileChannel`。

## 与 Stream API 区分

| | IO 流 | `java.util.stream.Stream` |
| --- | --- | --- |
| 用途 | 读写外部数据（文件、网络） | 对内存集合做函数式处理 |
| 典型操作 | `read` / `write` | `filter` / `map` / `collect` |
| 是否消耗资源 | 是，需关闭 | 否（数据源是集合时） |

名字都叫「流」，但完全不是一回事；IO 章节讲的是本节内容。

## 实践要点

- 文本一律指定 `Charset`（推荐 `StandardCharsets.UTF_8`），别依赖默认编码
- 加缓冲（`Buffered*` 或足够大的 `byte[]`）显著提升 IO 性能
- 二进制用字节流，文本用字符流
- 复制文件：字节流循环、`Files.copy`、或 `FileChannel.transferTo` 均可
